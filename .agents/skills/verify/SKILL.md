# verify — 代码完整性验证

按顺序执行以下检查，任何一步失败则停止并报告错误：

## 执行步骤

1. **编译检查**
   ```bash
   cargo check --workspace
   ```
   确认所有 workspace crate 可编译。

2. **静态分析**（如果 clippy 可用）
   ```bash
   cargo clippy --workspace -- -D warnings
   ```
   将 clippy 警告视为错误。如果 clippy 未安装，跳过此步并提示用户安装。

3. **运行测试**
   ```bash
   cargo test --workspace
   ```
   运行所有 workspace 成员的单元测试和集成测试。

## 输出格式

以简洁的表格报告每一步的结果：

| 检查项 | 结果 | 备注 |
|--------|------|------|
| 编译检查 | ✅ / ❌ | 如有错误列出首个错误 |
| 静态分析 | ✅ / ❌ / ⏭️ | 警告数量或跳过原因 |
| 测试 | ✅ / ❌ | 通过/失败/跳过的测试数 |

## 注意事项

- 在项目根目录执行，不需要 cd 到子目录
- 如果某一步失败，仍然继续执行后续步骤，让用户看到全部问题
- 失败时给出具体修复建议，而非泛泛的"请检查代码"
