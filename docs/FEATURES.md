# Hit 功能特性

> **设计理念**：Hit 的目标是在 Rust 原生实现中，将 Scoop 的使用体验提升到现代 CLI 工具的水准。
> 以下功能目前通过 PowerShell 包装函数（`ref/Microsoft.PowerShell_profile.ps1`）临时验证可行性，
> 最终将用 Rust 原生实现内置于 Hit 中。

---

## 1. 命令简写系统

在 Rust CLI 中内置子命令简写，无需借助 shell 包装：

| 完整命令 | 简写 | 说明 |
|----------|------|------|
| `hit install` | `hit i` | 安装软件包 |
| `hit search` | `hit s` | 搜索软件包 |
| `hit status` | `hit st` | 查看软件状态 |
| `hit update` | `hit u` | 更新软件包 |
| `hit uninstall` | `hit rm` | 卸载软件包 |
| `hit list` | `hit ls` | 列出已安装软件 |
| `hit bucket` | `hit b` | 管理软件源 |
| `hit cleanup` | `hit c` | 清理缓存 |

## 2. 交互式搜索安装 — `si`

基于 **fzf** 的交互式搜索安装工具，是 Hit 的核心体验特性。

### 设计思路

`install` 和 `search` 命令保持传统的命令行模式可用，但每次执行时会显示一条提示：

```
💡 提示：推荐安装 fzf 后使用 hit si <关键词> 体验交互式安装（此提示可永久关闭）
```

用户可以通过 `hit config set si_tip false` 永久关闭此提示。

### 用法

```
hit si <关键词>
```

### 工作流程

1. 在所有已注册 Bucket 中搜索匹配的软件包
2. 按包名分组聚合，显示 名称 / 版本 / 来源 / 可执行文件
3. 通过 fzf 呈现表格界面（自动列宽对齐、截断处理）
4. Enter → 自动安装所选包；Esc → 取消
5. 同名软件存在于多个 Bucket 时，第二次 fzf 让用户选择来源

### 界面示意

```
> fzf 搜索窗口
  git       2.40.0  [main,extras]  bin/git.exe
  python    3.12.0  [sdk]         bin/python.exe
```

### 依赖

- **fzf**（推荐）：提供交互式模糊搜索界面
- 若 fzf 未安装，`si` 命令降级为普通 `search` + 提示用户安装 fzf

## 3. 零污染安装

- 所有软件安装在用户目录 `~/.hit/apps/`，无需管理员权限
- 不写注册表、不修改系统文件
- **便携化**：解压即用，卸载即删

## 4. Shim 代理机制

- 在 `~/.hit/shims/` 生成轻量级代理 exe（~200KB）
- 加入 PATH 后自动转发命令到实际程序
- 版本切换时只需更新符号链接指向

## 5. SDK 多版本管理

- 支持 JDK、Python、Node.js 等 SDK 多版本共存
- `current` 符号链接切换当前激活版本
- 版本约束语法：`@latest`、`@stable`、`@^3.12`、`@3.12.0`

## 6. 深度卸载

- 扫描注册表卸载信息（`winreg`）
- 并行残留文件扫描（`walkdir` + `rayon`）
- 清理注册表键值、服务、计划任务
- 强制进程终止（`windows-rs` API）

## 7. 事务性安装

- 原子操作：任意阶段失败自动回滚
- 下载 → 校验哈希 → 解压 → 安装 → 生成 Shim 全链路保护
- 使用临时目录 + `std::fs::rename` 原子移动

## 8. 依赖解析

- Manifest 中声明依赖关系（版本约束、可选/必需）
- 依赖图检测循环依赖和版本冲突
- 先安装依赖，再安装主包

## 9. Bucket 优化

- **三层索引**：全局索引 → Bucket 缓存 → 源仓库
- **优先级系统**：main(100) > sdk(50) > extras(30)
- **软件别名**：`py` → `python`
- **安装前预览**：版本、大小、依赖、来源

## 10. 健康检查

- 定期校验文件完整性和哈希匹配
- 自动修复损坏（`hit repair`）
- 检查 Shim 指向是否正确

## 11. 镜像源管理

- 内置中国镜像（清华、阿里、UCloud）
- 速度测试与自动切换
- 区域感知：选择最近镜像

## 12. 软件束（Bundle）

- 一键安装多个软件（如 `dev-environment` 束）
- 支持导出/导入束配置
- 适合团队标准化开发环境

## 13. 沙盒环境（Shadow）

- 隔离运行时环境（独立 persist、独立环境变量）
- 类似 Python virtualenv 但通用
- 安全运行未知软件

## 14. 生命周期管理

- **归档**：将旧版本移至外部存储
- **孤立文件清理**：扫描无主残留文件
- **去重**：跨软件重复文件硬链接化
- **自动清理**：N 天未使用的版本自动删除

## 15. 运行时监控

- `hit top` — 实时软件资源占用
- `hit ps` — 进程树查看
- `hit trace` — 文件访问跟踪

## 16. 插件系统

- Lua 脚本引擎（`mlua`）
- 插件钩子：安装前/后、卸载前/后
- 自定义子命令

## 17. 其他

- **命令简写**：内置 `hit i` / `hit s` / `hit u` 等简写
- **配置同步**：跨设备同步配置和已安装列表
- **开发模式**：本地目录安装，文件变化自动重载
- **备份与恢复**：完整配置备份
- **增量更新**：仅下载差异部分，减少带宽
- **环境诊断**：`hit doctor` 一键排查问题
