# Hit 项目实现 TODO 清单

## 🎯 目标：将 Scoop 重构为 Rust，完全兼容 Scoop Bucket

---

## 架构设计概述

### Scoop PowerShell 模块分析

| Scoop 模块           | 功能描述                     | 对应 Rust Crate   |
| -------------------- | ---------------------------- | ----------------- |
| `lib/core.ps1`       | 基础工具、配置管理、路径处理 | `hit-common`      |
| `lib/manifest.ps1`   | Manifest 解析与验证          | `hit-manifest`    |
| `lib/install.ps1`    | 安装流程控制                 | `hit-core`        |
| `lib/database.ps1`   | SQLite 数据库缓存            | `hit-store`       |
| `lib/buckets.ps1`    | Bucket 仓库管理              | `hit-bucket`      |
| `lib/download.ps1`   | 文件下载                     | `hit-downloader`  |
| `lib/decompress.ps1` | 解压功能                     | `hit-compression` |
| `lib/depends.ps1`    | 依赖解析                     | `hit-core`        |
| `libexec/*.ps1`      | 命令实现                     | `hit-cli`         |
| `supporting/shims`   | Shim 代理                    | `hit-shim`        |

### Rust 项目结构设计

```
crates/
├── hit-cli/           # CLI 命令行接口（替代 libexec/*.ps1）
│   ├── src/
│   │   ├── main.rs
│   │   └── commands/
│   │       ├── install.rs
│   │       ├── uninstall.rs
│   │       ├── list.rs
│   │       ├── search.rs
│   │       ├── info.rs
│   │       ├── update.rs
│   │       └── bucket.rs
│   └── Cargo.toml
├── hit-core/          # 核心业务逻辑（替代 lib/install.ps1, depends.ps1）
│   ├── src/
│   │   ├── lib.rs
│   │   ├── installer.rs      # 安装流程控制器
│   │   ├── uninstaller.rs    # 卸载流程控制器
│   │   ├── dependency.rs     # 依赖解析器
│   │   ├── shim_manager.rs   # Shim 管理器
│   │   ├── persist.rs        # 持久化管理
│   │   ├── transaction.rs    # 事务管理器
│   │   └── hooks.rs          # 钩子脚本执行
│   └── Cargo.toml
├── hit-manifest/      # Manifest 解析（替代 lib/manifest.ps1）
│   ├── src/
│   │   ├── lib.rs
│   │   ├── parser.rs         # JSON 解析器
│   │   ├── validator.rs      # 验证器
│   │   ├── schema.rs         # Schema 定义
│   │   └── variables.rs      # 变量替换（如 $version, $architecture）
│   └── Cargo.toml
├── hit-bucket/        # Bucket 管理（替代 lib/buckets.ps1）
│   ├── src/
│   │   ├── lib.rs
│   │   ├── manager.rs        # Bucket 管理器
│   │   ├── git_client.rs     # Git 仓库操作
│   │   ├── index.rs          # 软件索引
│   │   └── registry.rs       # 已知 Bucket 注册表
│   └── Cargo.toml
├── hit-downloader/    # 下载模块（替代 lib/download.ps1）
│   ├── src/
│   │   ├── lib.rs
│   │   ├── http.rs           # HTTP 下载器
│   │   ├── cache.rs          # 缓存管理
│   │   ├── hash.rs           # 哈希校验
│   │   └── aria2.rs          # Aria2 集成（可选）
│   └── Cargo.toml
├── hit-compression/   # 解压模块（替代 lib/decompress.ps1）
│   ├── src/
│   │   ├── lib.rs
│   │   ├── zip.rs            # ZIP 解压
│   │   ├── sevenz.rs         # 7z 解压
│   │   ├── tar.rs            # TAR 解压
│   │   └── installer.rs      # 安装程序处理（exe/msi）
│   └── Cargo.toml
├── hit-store/         # 数据存储（替代 lib/database.ps1）
│   ├── src/
│   │   ├── lib.rs
│   │   ├── db.rs             # SQLite 数据库
│   │   ├── models.rs         # 数据模型
│   │   └── migration.rs      # 数据库迁移
│   └── Cargo.toml
├── hit-shim/          # Shim 代理（替代 supporting/shims）
│   ├── src/
│   │   └── main.rs           # 独立可执行文件
│   └── Cargo.toml
├── hit-windows/       # Windows 特定功能（替代 lib/system.ps1）
│   ├── src/
│   │   ├── lib.rs
│   │   ├── process.rs        # 进程管理
│   │   ├── registry.rs       # 注册表操作
│   │   ├── fs.rs             # 文件系统操作
│   │   ├── shell.rs          # Shell 集成
│   │   └── uac.rs            # UAC 提权
│   └── Cargo.toml
└── hit-common/        # 公共工具（替代 lib/core.ps1）
    ├── src/
    │   ├── lib.rs
    │   ├── config.rs         # 配置管理
    │   ├── paths.rs          # 路径处理
    │   ├── log.rs            # 日志系统
    │   ├── error.rs          # 错误类型定义
    │   └── utils.rs          # 通用工具函数
    └── Cargo.toml
```

### Rust 优势利用

| Rust 特性  | 应用场景             | 带来的好处                 |
| ---------- | -------------------- | -------------------------- |
| 类型安全   | Manifest 数据结构    | 编译期检查，避免运行时错误 |
| 所有权系统 | 文件操作、资源管理   | 自动资源清理，无内存泄漏   |
| 并发支持   | 并行下载、索引构建   | 提升性能                   |
| 零成本抽象 | 模块化设计           | 代码复用，无运行时开销     |
| 宏系统     | 命令行解析、错误处理 | 减少样板代码               |
| Cargo 生态 | 依赖管理、构建系统   | 简化开发流程               |

## 📝 Rust 重构关键设计决策

### 1. Manifest 解析设计
- 使用 `serde` 进行 JSON 解析，支持 Scoop 的所有字段
- 变量替换使用自定义解析器，支持 `$version`, `$architecture`, `$url` 等变量
- 验证器使用 `schemars` 生成 JSON Schema 验证

### 2. 事务性安装设计
- 使用 RAII 模式管理事务状态
- 原子操作使用 Windows 的 `MoveFileEx` API
- 失败回滚通过删除临时目录实现

### 3. 并发设计
- 使用 `rayon` 进行并行下载
- 使用 `tokio` 进行异步 I/O
- Bucket 索引构建使用多线程并行处理

### 4. 错误处理设计
- 使用 `thiserror` 定义统一错误类型
- 使用 `anyhow` 进行错误传播
- 提供详细的错误信息和修复建议

### 5. 性能优化
- 使用 `bytes` crate 进行高效内存管理
- 使用 `lru` crate 实现缓存
- 使用 `memmap` 进行大文件操作
