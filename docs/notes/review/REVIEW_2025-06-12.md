## 值得注意的技术细节

1. **env 测试隔离**：`#[should_panic]` 测试会在 panic  unwind 时丢失 `restore_env` 调用，污染后续并行测试。改用 `std::panic::catch_unwind` 包裹被测调用，保证 env 在任何路径下都被恢复。
2. **跨模块 env 锁**：`paths::tests` 与 `session::tests` 共享 `pub(crate) static ENV_LOCK`，避免 session 测试在 paths 测试清空 env 时并发调用 `root_path()` 导致 panic。
3. **Poisoned Mutex 恢复**：`acquire_env_lock()` 使用 `lock().unwrap_or_else(|e| e.into_inner())`，即使 `catch_unwind` 路径中 MutexGuard 未正常释放也能恢复。
4. **Rust 2024 `unsafe` env**：`std::env::set_var` / `remove_var` 在 2024 edition 中被标记为 unsafe，测试中以 `unsafe { ... }` 包裹并附加 SAFETY 注释。
5. **Let chains**：`paths::root_path()` 使用 `if let Ok(p) = var(...) && !p.is_empty()` 简化嵌套（2024 edition 稳定特性）。
