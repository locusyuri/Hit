### 1️⃣ LinkMode 简化：只用 Junction

**决策**：删除 `LinkMode` 枚举，不再保留 Symlink / Hardlink 选项。仅使用 Junction。

**依据**（来自 Scoop 源码分析 `ref/Scoop/lib/install.ps1`）：

Scoop 在以下两处使用链接：

| 用途 | Scoop 使用的方式 |
|------|-----------------|
| `apps/<app>/current` → 版本目录 | **Junction**（`New-DirectoryJunction`） |
| `persist/` 下的数据 | 目录用 **Junction**，文件用 **HardLink** |

Scoop 的 `NO_JUNCTION` 配置项效果是让 `current` 退化为从 `db.json` 读取版本号，**而不是**切换到 Symlink。Scoop 本身没有 Symlink 选项。

Junction 的优势：
- ❌ 不需要管理员权限 / 开发者模式
- ✅ 支持跨驱动器
- ✅ 足够满足版本切换需求

**待修改的代码位置**：
- `crates/hit-common/src/config.rs`：删除 `LinkMode` 枚举，相关字段改为 `no_junction: bool`
- `crates/hit-test-utils/src/lib.rs`：更新 `mock_config()` 函数
- `docs/TECH_STACK.md`：更新技术栈说明

### 2️⃣ 路径回退策略

**决策**：参照 Scoop 的路径解析逻辑改进 `root_path()` 的 fallback 链。

**Scoop 的做法**（`ref/Scoop/lib/core.ps1:1291`）：

```powershell
$scoopdir = $env:SCOOP,                          # 1) 环境变量 SCOOP
            (get_config ROOT_PATH),              # 2) 配置文件 ROOT_PATH
            "$PSScriptRoot\..\..\..\..",         # 3) 脚本相对路径（开发用）
            "$HOME\scoop"                        # 4) 默认 %USERPROFILE%\scoop
            | Where-Object { $_ } | Select-Object -First 1
```

**Hit 应实现的 fallback 链**（优先级从高到低）：

```
1. 环境变量 HIT_ROOT          # 用户显式指定
2. 配置文件 root_path         # 配置文件中的路径
3. %USERPROFILE%\.hit         # 默认路径
4. 若 USERPROFILE 和 HOME 都未设置 → panic 并报错"无法确定用户目录"
```

同时保留 `SCOOP` 环境变量作为兼容读取（如果设置了 `SCOOP` 但未设置 `HIT_ROOT`，也可以使用）。

### 3️⃣ 安装分发方式：需要 PowerShell 安装脚本

**决策**：参照 Scoop 的 `irm get.scoop.sh | iex` 模式，提供一个 PowerShell 安装脚本。

**Scoop 的做法**：
- 用户执行 `irm get.scoop.sh | iex`（从远程加载并执行安装脚本）
- 安装脚本会：
  1. 检查环境（PowerShell 版本、执行策略等）
  2. 下载 `scoop.zip` 到临时目录
  3. 解压到用户指定的目录（默认 `~/.scoop/` → 最终是 `~/scoop`）
  4. 将 Scoop 自身通过 `scoop install scoop` 安装（即用 Scoop 装 Scoop 自己）
  5. 注册 shims 目录到 PATH

**Hit 的建议安装脚本流程**：

```powershell
# install-hit.ps1（伪代码）
# 用法：irm https://get.hit.sh | iex
# 或：.\install-hit.ps1 [-Path C:\Users\me\.hit] [-Mirror tuna]

1. 检查环境
   - PowerShell 5+
   - 执行策略允许（自动 Set-ExecutionPolicy RemoteSigned -Scope CurrentUser）
   - 检查是否已安装（已安装则提示更新或退出）

2. 确认安装路径
   - 参数 -Path，默认 $env:HIT_ROOT ?? "$HOME\.hit"
   - 自动创建目录

3. 下载 hit.exe
   - 从 GitHub Releases 下载对应架构的 zip
   - 解压到安装路径

4. 初始化目录结构
   - 创建 apps/ shims/ cache/ persist/ buckets/ logs/
   - 写入默认 config.json
   - 写入默认 scoop_bucket.json（内置 main bucket）

5. 注册 PATH
   - 将安装路径（含 shims/ 子目录）添加到 HKCU\Environment PATH
   - 广播 WM_SETTINGCHANGE

6. 一次性自举
   - 运行 `hit install scoop` 添加 main/extras bucket（可选）
   - 或下载 Scoop 官方 bucket 的索引快照

7. 完成提示
```

**关于路径可配置性**：
- 支持 `-Path` 参数自定义安装路径
- 环境变量 `HIT_ROOT` 可覆盖
- 配置文件中的 `root_path` 也可修改（但仅对已安装后的运行生效）
- 三者优先级：`HIT_ROOT` 环境变量 > `-Path` 参数 > 默认 `~/.hit`

**决定的理由**：
- Scoop 的安装模式已被广泛接受，用户习惯熟悉
- PowerShell 脚本无需编译、无需额外运行时
- 可以处理 PATH 注册、UAC 提权等 Rust exe 不方便做的事
- 后续升级可以通过 `hit update` 内部完成（Rust 实现），安装脚本仅用于**首次安装**
